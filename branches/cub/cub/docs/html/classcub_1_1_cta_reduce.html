Detailed Description

template<typename T, int CTA_THREADS>
class cub::CtaReduce< T, CTA_THREADS >

The CtaReduce type provides variants of parallel reduction across threads within a CTA.

.

Overview

A reduction (or fold) uses a binary combining operator to compute a single aggregate from a list of input elements. The parallel operations exposed by this type assume n-element lists that are partitioned evenly across CTA_THREADS threads, with threadi owning the ith element (or ith segment of consecutive elements). To minimize synchronization overhead, these operations only produce a valid cumulative aggregate for thread0.
Template Parameters
TThe reduction input/output element type
CTA_THREADSThe CTA size in threads

Important Features and Considerations

  • Supports non-commutative reduction operators.
  • Supports partially-full CTAs (i.e., high-order threads having undefined values).
  • Very efficient (only one synchronization barrier).
  • Zero bank conflicts for most types.
  • After any operation, a subsequent CTA barrier (__syncthreads()) is required if the supplied CtaReduce::SmemStorage is to be reused/repurposed by the CTA.
  • The operations are most efficient (lowest instruction overhead) when:
    • The data type T is a built-in primitive or CUDA vector type (e.g., short, int2, double, float2, etc.) Otherwise the implementation may use memory fences to prevent reference reordering of non-primitive types.
    • CTA_THREADS is a multiple of the architecture's warp size
    • Every thread has a valid input (i.e., unguarded reduction)

Algorithm

These parallel reduction variants have O(n) work complexity and are implemented in three phases:
  1. Sequential reduction in registers (if threads contribute more than one input each). Each thread then places the partial reduction of its item(s) into shared memory.
  2. A single-warp performs a raking upsweep across partial reductions shared each thread in the CTA.
  3. A warp-synchronous Kogge-Stone style reduction within the raking warp to produce the total aggregate.

Examples

  • Example 1: Simple reduction
    #include <cub.cuh>
    template <int CTA_THREADS>
    __global__ void SomeKernel(...)
    {
    // Parameterize CtaReduce for use with CTA_THREADS threads on type int
    typedef cub::CtaReduce<int, CTA_THREADS> CtaReduce;
    // Declare shared memory for CtaReduce
    __shared__ typename CtaReduce::SmemStorage smem_storage;
    // A segment of four input items per thread
    int data[4];
    ...
    // Compute the CTA-wide sum for thread0
    int aggregate = CtaReduce::Reduce(smem_storage, data);
    ...
    }

  • Example 2: Guarded reduction
    #include <cub.cuh>
    template <int CTA_THREADS>
    __global__ void SomeKernel(..., int num_elements)
    {
    // Parameterize CtaReduce for use with CTA_THREADS threads on type int
    typedef cub::CtaReduce<int, CTA_THREADS> CtaReduce;
    // Declare shared memory for CtaReduce
    __shared__ typename CtaReduce::SmemStorage smem_storage;
    // Guarded load
    int data;
    if (threadIdx.x < num_elements) data = ...;
    // Compute the CTA-wide sum of valid elements in thread0
    int aggregate = CtaReduce::Reduce(smem_storage, data, num_elements);
    ...
