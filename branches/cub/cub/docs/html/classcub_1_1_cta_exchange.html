Detailed Description

template<typename T, int CTA_THREADS, int ITEMS_PER_THREAD>
class cub::CtaExchange< T, CTA_THREADS, ITEMS_PER_THREAD >

The CtaExchange type provides operations for reorganizing the partitioning of ordered data across CTA threads.

.

Overview

The operations exposed by CtaExchange allow CTAs to reorganize data items between threads, converting between (or scattering to) the following partitioning arrangements: -blocked arrangement. The aggregate tile of items is partitioned evenly across threads in "blocked" fashion with threadi owning the ith segment of consecutive elements. -striped arrangement. The aggregate tile of items is partitioned across threads in "striped" fashion, i.e., the ITEMS_PER_THREAD items owned by each thread have logical stride CTA_THREADS between them.
Template Parameters
TThe data type to be exchanged.
CTA_THREADSThe CTA size in threads.
ITEMS_PER_THREADThe number of items partitioned onto each thread.

Important Features and Considerations

  • After any operation, a subsequent CTA barrier (__syncthreads()) is required if the supplied CtaExchange::SmemStorage is to be reused/repurposed by the CTA.
  • Zero bank conflicts for most types.

Algorithm

Regardless of the initial blocked/striped arrangement, CTA threads scatter items into shared memory in blocked, taking care to include one item of padding for every shared memory bank's worth of items. After a barrier, items are gathered in the desired blocked/striped arrangement.

A CTA of 16 threads performing a conflict-free blocked gathering of 64 exchanged items.
