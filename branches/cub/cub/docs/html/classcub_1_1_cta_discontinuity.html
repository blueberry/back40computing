Detailed Description

template<typename T, int CTA_THREADS>
class cub::CtaDiscontinuity< T, CTA_THREADS >

The CtaDiscontinuity type provides operations for flagging discontinuities within a list of data items partitioned across CTA threads.

.

Overview

The operations exposed by CtaDiscontinuity allow CTAs to set "head flags" for data elements that are different from their predecessor (as specified by a binary boolean operator).
Template Parameters
TThe data type to be exchanged.
CTA_THREADSThe CTA size in threads.

Important Features and Considerations

  • After any operation, a subsequent CTA barrier (__syncthreads()) is required if the supplied CtaDiscontinuity::SmemStorage is to be reused/repurposed by the CTA.
  • Zero bank conflicts for most types.

Sets discontinuity flags for a tile of CTA items, for which the first item has no reference (and is always flagged).

Assuming a blocked arrangement of elements across threads, flagsi is set non-zero for item inputi when scan_op(previous-item, inputi) is true (where previous-item is either inputi-1, or inputITEMS_PER_THREAD-1 in the previous thread). Furthermore, flagsi is always non-zero for input0 in thread0. The last tile item of the last thread is also returned to thread0.

The last_tile_item is undefined in threads other than thread0.

A subsequent __syncthreads() CTA barrier is required if the supplied smem_storage is to be reused/repurposed by the CTA.

Sets discontinuity flags for a tile of CTA items.

Assuming a blocked arrangement of elements across threads, flagsi is set non-zero for item inputi when scan_op(previous-item, inputi) is true (where previous-item is either inputi-1, or inputITEMS_PER_THREAD-1 in the previous thread). For thread0, item input0 is compared against /p tile_prefix. The last tile item of the last thread is also returned to thread0.

The tile_prefix and last_tile_item are undefined in threads other than thread0.

A subsequent __syncthreads() CTA barrier is required if the supplied smem_storage is to be reused/repurposed by the CTA.
